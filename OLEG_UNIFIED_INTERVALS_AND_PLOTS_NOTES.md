# Что изменено

1. Добавлен единый загрузчик рынка из БД:
   - `fetch_unified_market_window(...)`
   - база = `candles`
   - `OI` подтягивается по `LEFT JOIN open_interest ON oi.ts = candle.open_time`
   - `CVD` берётся из той же строки свечи

2. Pump screener переведён на одинаковые интервалы из БД:
   - расчёты `OI/CVD` теперь идут только в том же `interval`, что и свечи сигнала
   - убраны fallback-интервалы `5m/15m/1h/4h` для flow-расчётов

3. Plot переведён на тот же unified dataset:
   - цена, OI и CVD строятся по одной и той же временной оси
   - нижние панели больше не должны смещаться из-за отдельных LIMIT по OI/CVD

4. Исправлен `trend_mode: soft`:
   - `soft` теперь не отбрасывает кандидата как `hard`
   - тренд-фейлы пишутся в диагностику

5. Исправлен BUY early/confirmed:
   - `confirmed` требует жёсткий OI + CVD и свой score
   - `early` больше не требует тот же жёсткий OI/CVD как confirmed

6. В заголовок графика добавлено имя скринера.

# Что важно помнить

- Если `1h` OI в БД заполнен неполно, график и расчёты будут честно показывать пропуски.
- Это уже проблема наполнения таблицы `open_interest`, а не plot/screener кода.
